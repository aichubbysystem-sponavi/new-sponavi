### 変更内容（具体的に）

#### src/lib/detect-language.ts — 言語判定の根本バグ修正
- GBP口コミ "(Translated by Google) [翻訳] (Original) [原文]" のフォーマットで、(Original)以降を削除して翻訳テキストを判定していた
- 修正: (Original)がある場合はその後ろ（原文テキスト）を判定対象にする
- これにより韓国語・タイ語・ベトナム語等がGoogleに日本語翻訳されても正しい原文言語が検出される
- みそラーメンのよし乃: 修正前3言語(日・英・中) → 修正後7言語(日・英・韓・中・仏・タイ・越)

#### src/app/api/report/analyze/route.ts — AI総評に言語データ追加
- analyzeWithClaude() と tryAnalyze() に langStatsText?: string パラメータ追加
- POST handler内: Supabaseからreviews.commentを直接取得（DB生コメント）してdetectLanguageで言語集計生成
- プロンプトに【口コミ言語別集計】セクションを口コミテキスト直後に挿入
- commentsルールに「集計に含まれない言語を推測で記述することは厳禁」追加（langStatsTextがある時のみ）
- デバッグログ追加: [analyze] ${shop.name}: langStats=X言語検出

#### src/app/api/cron/monthly-analysis/route.ts — cronにも言語データ追加
- 口コミ言語別集計を生成（reviews.commentをそのままdetectLanguageに渡す）
- プロンプトに【口コミ言語別集計】セクション + 推測記述禁止ルール追加

#### src/app/api/report/review-language-stats/route.ts — API高速化
- shopNames→shop_id逆引き（reviewsテーブル2回検索+ilike検索、27行）を完全削除
- shopNamesがあればshop_nameで直接検索、shopIdsがあればshop_idで検索（5行に簡素化）
- DB往復が半減し表示速度が改善（約10秒→数秒）

#### src/app/report/[shopId]/client.tsx — レポートUI改善（8項目）
1. 最小フォントサイズ16px統一: fontSize 9〜15px を全て16pxに引き上げ（133箇所）、fontSize: "11px"(文字列) → "16px"
2. KPIサマリーグリッド: gridTemplateColumns repeat(4,1fr) → repeat(3,1fr)
3. 左右padding統一: slideBarStyle/slideBodyStyle/ヘッダー/メタ情報バー/多地点順位の全5箇所 36px→9px
4. チャート高さ縮小: buildStackedOptions()にaspectRatio: 2.2追加
5. チャートテーブルの改行防止: 全ラベルセル（月/モバイル/PC/Web/ルート/通話/メニュー/予約/合計）にwhiteSpace: "nowrap"追加
6. キーワード順位推移テーブルはみ出し: overflow: hidden → overflowX: auto、セルpadding 12px→6px、テーブルminWidth: 700
7. 検索語句テーブル15位はみ出し: slideBody padding 28px→10px、thStyle padding 10px→6px、tdセル padding 7px→4px
8. 最新月背景色: #f0f4ff → #cfffE3（薄緑）
9. AIコメント本文: fontSize 16→17px、コンテナpadding 28px 32px→16px 20px、overflow: hidden維持（PDF対応）

### 注意点・ハマったこと
- detectLanguageの(Original)処理が逆だった: 原文を捨てて翻訳テキストを判定していた。GBPの口コミフォーマットを正確に理解していないと気づけないバグ
- fetchReviewsの加工済みcomment vs DB生コメント: fetchReviewsは(Original)以降を抽出（原文側）、detectLanguageは(Original)以降を除去（翻訳側）で判定対象が逆になる。review-language-stats APIと同じ処理パス（DB生コメント→detectLanguage）で統一
- shop.idの正体: review-analysis/page.tsxのshopsはuseShop()（Go API由来）→ shop.id = Go API ID。ShopListItem.id = 店舗名（spreadsheet.ts:796）はレポート管理画面の別データソース
- review-language-stats APIのshop_id逆引き: reviewsテーブルを2回検索していたのが10秒遅延の主因。shop_nameで直接検索に変更で解消
- AIコメントのoverflow: auto（スクロール）はPDFダウンロード時にスクロール内の内容が切れるためhiddenに戻した
