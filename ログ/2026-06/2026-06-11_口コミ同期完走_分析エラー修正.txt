### 注意点・ハマったこと
- Chromeバックグラウンドタブ制限: setTimeoutが最大1分間隔まで遅延 → 最初の店舗で3時間以上ハングした
  → AbortSignal.timeout()もブラウザネイティブだが同様に遅延される可能性あり
  → Promise.raceの組み合わせで多層防御、かつ**タブをアクティブにしたまま**実行する運用で回避
  → タブを別ウィンドウに分離すればバックグラウンド制限を受けない
- axiosのtimeoutオプションだけではTCP接続ハングを検知できない
  → AbortSignal.timeout()でfetch自体を強制切断する必要あり
- Vercelのエラーレスポンス形式: {error: "string"}ではなく{error: {code, id, message}}オブジェクト
  → フロントでJSXに{error}をそのまま渡すとReact error #31（Objects are not valid as React child）
  → safeStr()で防御
- 同名店舗問題: .eq("name", shopName)が50箇所以上あるが、shops_name_unique制約で現在は全店舗ユニーク名
  → Go API IDベースに移行するにはspreadsheet.ts/全APIルート/レポートURL構造の全面改修が必要（数日規模）
  → 現状維持が合理的、新規同名発生時のみサフィックス付与で対応
